# 反向传播演示页双阶段教学与可视化增强设计

## 背景
当前反向传播教学演示已经具备单步播放、自动播放、网络图和 loss 历史等基础能力，但仍有三个明显的教学痛点：

1. 样本数据对初学者仍偏抽象，进入页面后不容易立即理解“这次在学什么”。
2. 页面能看到网络内部的权重和梯度，却不够直观地展示“目标函数是什么”“loss 为什么会下降”。
3. 网络图中的交叉连线权重标签会互相遮挡，影响讲解时的可读性。

本次设计目标是在不推翻现有页面结构的前提下，把演示改造成“先讲单样本细节，再看整体训练趋势”的双阶段教学页，并补足目标函数、损失函数和梯度下降路径的可视化。

## 目标

### 教学目标
- 让初学者先看懂一次前向传播、loss、反向传播、参数更新的完整链路。
- 让观众随后看到：单步更新不是孤立动作，而是在持续推动 loss 下降。
- 明确区分“网络内部的反向传播过程”和“外部优化目标的下降趋势”。

### 界面目标
- 保留现有左右双栏的整体骨架，避免完全重学操作方式。
- 在同一页同时呈现网络图与函数/损失图，但保持默认视图清晰、重点明确。
- 消除交叉权重标签遮挡问题，提升投屏讲解时的可读性。

## 非目标
- 不把页面扩展成完整的数学推导课件。
- 不引入新的复杂网络结构或新的训练算法。
- 不追求数值上严格模拟真实多样本回归训练的全部细节，重点是教学上的可理解性。

## 方案概览
采用“同页双视图联动”的实现方案。

- 左侧保留控制面板、网络图和流程/趋势面板，继续承载“网络内部发生了什么”。
- 右侧在保留关键说明信息的基础上，加入两个新图：目标函数图、损失与梯度下降图，承载“优化目标是什么、训练为何有效”。
- 数据上使用双模式：
  - 网络反向传播讲解使用极简二值样本。
  - 函数可视化单独使用一维连续目标函数。

这样可以避免把“链式法则教学”和“函数逼近教学”硬塞进同一套数据表示里，同时又通过共同的 loss 概念把两者串起来。

## 数据设计

### 一、单样本拆解数据
保留小型 2 -> 2 -> 1 网络，默认使用极简样本集合进行讲解。

建议展示的核心样本：
- `sample-a`: `(0, 0) -> 0`
- `sample-b`: `(1, 0) -> 1`

可选保留补充样本用于 epoch 模式，但默认教学焦点锁定在上述两个样本上。原因：
- 输入与目标都足够直观，不需要额外解释模式含义。
- 样本之间差异鲜明，便于说明“为什么同一套权重会对不同输入产生不同输出”。
- 保持与现有 2 输入结构兼容，减少模型和画布改动范围。

### 二、目标函数可视化数据
新增一套独立于网络图的连续函数演示数据，使用一维输入 `x` 与一条简单目标函数曲线。

目标函数设计原则：
- 单调、平滑、视觉上容易理解。
- 不追求复杂非线性，只强调“目标值是怎样随输入变化的”。
- 可在单一样本模式下标出“当前输入 x 的目标点”和“模型当前预测点”。

推荐使用简单线性或轻微弯曲但连续的函数形式，例如接近 `target = x` 的标准化曲线。这样观众能快速接受“模型正在逼近一条目标关系”。

### 三、loss 数据
loss 继续保留现有按步骤和按 epoch 采样的历史记录，但会分为两种展示语义：
- `sample` 模式：强调当前样本下的一步更新前后变化。
- `epoch` 模式：强调多轮训练中的总体下降趋势。

## 页面结构设计

### 左侧主区
保留现有三段结构：
- 控制面板
- 网络图
- 流程/趋势面板

调整重点：
- `sample` 模式下，底部面板更突出步骤流程。
- `epoch` 模式下，底部面板更突出 loss 历史。

### 右侧辅区
由原本单一指标说明区改为“说明卡片 + 双图卡片”的结构：
- 当前步骤说明
- 当前样本与关键指标
- 目标函数图
- 损失函数 / 梯度下降图
- 训练日志

这样右侧从“纯文本说明”升级为“文字 + 图形证据”的解释区。

## 图表设计

### 一、目标函数图
用途：回答“模型到底要逼近什么”。

图中元素：
- 目标函数曲线
- 当前输入位置的垂直辅助线
- 当前样本目标点
- 模型预测点
- 简短图例或标题说明

展示重点：
- 在 `sample` 模式下，清楚看到预测点与目标点之间的误差。
- 在 `epoch` 模式下，弱化单点细节，只保留函数背景和更稳态的说明。

### 二、损失函数 / 梯度下降图
用途：回答“参数更新为什么有意义”。

图中元素：
- loss 历史折线
- 当前迭代位置高亮点
- 单步梯度下降提示箭头或更新前后小标记
- 纵轴方向上的“loss 更小 / 更大”视觉暗示

展示重点：
- `sample` 模式下，用局部高亮强调“这一步在往更低 loss 方向移动”。
- `epoch` 模式下，用完整折线强调“多轮训练整体在下降”。

严格来说，这里显示的是教学化的“梯度下降过程示意”，而不是完整二维参数地形图。这样可以在有限版面中保持清晰。

## 网络图优化设计

### 一、权重标签避让
当前权重标签直接放在连线中点，交叉边会重叠。调整方案：
- 标签不再压在线中点上，而是沿连线中段做法向偏移。
- 不同方向的连线使用不同偏移方向和偏移量。
- 标签改成带圆角背景的小胶囊，提高在亮色线条上的对比度。

### 二、节点布局微调
在不改变整体结构认知的前提下，轻微拉开隐藏层节点与输出节点的相对位置，减轻交叉区域视觉拥挤。

### 三、信息层级
通过颜色和描边区分三类信息：
- 前向传播高亮
- 反向传播高亮
- 权重数值标签

保证投影环境下也能快速识别当前讲解焦点。

## 交互与状态设计

### 保留现有模式
继续保留：
- `sample`：单样本逐步拆解
- `epoch`：整轮训练趋势

### 状态联动原则
- 控制面板继续驱动主状态机，不增加新的一级模式开关。
- 新增图表只消费现有状态和派生数据，尽量避免生成独立复杂状态。
- `sample` 模式优先展示局部细节。
- `epoch` 模式优先展示整体趋势。

### 默认体验
页面初始进入时：
- 默认网络为小型结构 `2 -> 2 -> 1`
- 默认模式为 `sample`
- 默认高亮首个核心样本
- 默认右侧展示“目标函数点差 + 当前一步 loss 解释”

## 组件与模型改造建议

### 模型层
在 `backpropDemoModel.ts` 中新增或调整：
- 更简化的教学样本数据定义
- 函数图采样点生成函数
- 当前预测点映射函数
- loss 历史点和高亮点所需的派生数据

模型层应尽量输出“可直接渲染的图表数据”，避免组件里散落数学细节。

### 组件层
建议新增一个专门的右侧可视化组件，例如：
- `BackpropFunctionPanel.tsx`

内部可再拆成两个子图，但优先保持文件职责集中，便于维护。

现有组件改造方向：
- `BackpropDemoPage.tsx`：重新组织右侧布局与派生数据。
- `BackpropNetworkCanvas.tsx`：处理权重标签避让与布局优化。
- `BackpropTimelinePanel.tsx`：增强 loss 趋势表达。
- `BackpropMetricsPanel.tsx`：缩减为更聚焦的文字说明卡，而不是承载全部右栏内容。

## 错误处理与鲁棒性
- 图表数据为空时，提供简短占位说明，不让 SVG 区域空白。
- 历史点不足时，允许折线退化为点或短线段。
- 模式切换或重置时，所有右侧图都应同步回到一致状态。
- 中英文文案统一清理，避免当前存在的乱码问题继续扩散。

## 测试策略

### 单元测试
- 验证教学样本集生成逻辑。
- 验证函数图采样点与当前预测点生成逻辑。
- 验证 loss 历史派生数据在短序列和长序列下的稳定性。

### 组件测试
- 验证页面渲染出目标函数图和损失图标题。
- 验证 `sample` / `epoch` 模式切换后，右侧说明重点发生变化。
- 验证网络图仍会显示权重标签，且关键说明元素存在。

### 回归关注点
- 现有单步执行、自动播放、暂停、重置不应失效。
- 小网络和中网络两种 preset 都应继续工作。
- loss 历史与日志追加逻辑不能被新图表破坏。

## 范围控制
本次实现只做教学清晰度增强，不做以下扩展：
- 不引入三维 loss 曲面。
- 不增加公式面板或可编辑公式系统。
- 不增加新的训练模式或自定义数据编辑器。

## 预期结果
完成后，页面应具备以下体验：
- 初学者能先用一个直观样本看懂一次反向传播。
- 观众能在同一页看到目标函数、预测偏差与 loss 下降之间的关系。
- 网络图中的权重标签在交叉区域不再互相遮挡。
- 整个演示在投屏和讲解场景下更清楚、更稳定。
