项目背景：
这是一个技术分享用前端，不是面向用户转化的产品官网。
观众主要是工程师，因此页面应该强调：
- 信息结构清晰
- 逻辑链明确
- 图解和流程表达强
- 少营销话术
- 少装饰性元素

分享主题逻辑：
全分享有一条主线：
“模型能力如何演进 -> 为什么会走向 Agent -> 工业实现如何落地”

因此三章关系是：
1. 大模型发展：解释背景和能力演进
2. Agent 框架选型：解释工程决策
3. Claude 工程解析：解释工业级实现的验证案例

页面语气：
- 偏理性、偏工程
- 结论导向
- 不要写成科普海报
- 不要写成市场宣传页

视觉风格：
- 深色、简洁、克制
- 有高级感，但不过度炫技
- 适合在会议室投屏
- 避免大量渐变噪声、玻璃态堆砌、复杂背景纹理

内容组织方式：
- 每章先有 overview
- 再有 stage-by-stage 详细拆解
- 每个 stage 都有：
  - 核心问题
  - 做了什么优化
  - 解决了什么
  - 还剩什么问题
  - 一个前端 demo

我更看重：
1. 结构清楚
2. demo 便于讲解
3. 效果美观，炫酷

我不看重：
1. 巨量第三方依赖
