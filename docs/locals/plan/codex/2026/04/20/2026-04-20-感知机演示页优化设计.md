# 感知机介绍区优化与独立高交互 Demo 页设计

## 1. 背景与目标

当前项目中的“感知机分类实验”主要以样本卡片和说明文字呈现，缺少平面几何上的直观展示，也没有把 XOR 问题作为单层感知机能力边界来完整展开。此次改造的目标分为两层：

1. 主页面中的感知机区域保持轻量、简洁、适合串讲，不让整页信息密度失控。
2. 通过新窗口打开一个独立的高交互 Demo 页，完整展示线性可分训练、XOR 单层失败、XOR 两层网络成功的全过程。

## 2. 用户确认后的范围

已确认的需求如下：

1. 采用“主页面轻讲解 + 独立高交互 Demo 页”的整体方案。
2. 独立演示内容通过新窗口打开，而不是在当前页内切路由。
3. 独立 Demo 页需要较强交互能力，支持切换数据集、调整训练参数、自动训练动画、单层与双层对比。
4. XOR 部分需要做到完整演示：
   - 单层感知机进行真实训练并稳定表现为无法解决。
   - 两层网络进行真实训练，并能展示训练过程和边界变化。

## 3. 推荐实现方案

### 3.1 主页面

主页面只改造现有“感知机分类实验”区域，不改变其它章节布局和风格。

改造后的感知机区域包含：

1. 一个小型二维坐标示意图，用于展示正负样本点和当前线性分界线。
2. 一段简洁说明，强调“感知机是在二维平面上学习一条线，把两类点分开”。
3. 一段对 XOR 的轻提示，明确说明“XOR 不是单层感知机能处理的问题”。
4. 一个“打开完整 Demo”按钮，点击后通过 `window.open` 打开独立新窗口。

主页面职责只承担“建立直觉”和“引导进入完整演示”，不在此处堆叠大量交互控件。

### 3.2 独立 Demo 页

独立 Demo 页采用适合投屏的大屏演示布局，承载完整交互和训练过程。页面包含三个核心场景：

1. 线性可分训练
2. XOR 单层感知机失败
3. XOR 两层网络成功

用户可在同一页面中切换场景，避免多页面跳转打断讲解节奏。

## 4. 页面结构设计

### 4.1 顶部控制栏

顶部区域用于集中控制整个演示流程，包含：

1. 页面标题与一句副标题。
2. 场景切换按钮：`线性可分`、`XOR 单层感知机`、`XOR 两层网络`。
3. 训练控制按钮：`单步训练`、`自动训练`、`暂停`、`重置`。
4. 参数调节控件：学习率、训练速度、必要时补充最大迭代轮数。

### 4.2 中央主画布

主画布是演示核心，负责展示：

1. 坐标轴与网格。
2. 样本点位置与类别颜色。
3. 当前决策边界。
4. 分类区域或高亮误分类点。
5. 在 XOR 两层网络场景中，额外展示隐藏层边界和最终组合区域。

主画布需要把几何直觉讲清楚，让观众能看到“边界如何移动”和“为什么单层不够”。

### 4.3 右侧讲解与指标面板

右侧面板负责补充当前场景的解释和状态信息，包括：

1. 当前权重、偏置、学习率。
2. 当前 epoch、误分类数、准确率或损失。
3. 当前场景的关键一句话讲解。
4. 在 XOR 单层场景下，明确提示“训练受限于表达能力，而不是训练轮数不够”。

### 4.4 底部训练趋势区

底部展示一条简洁的训练趋势图或状态时间线，用于帮助讲述：

1. 线性可分时误分类数逐步下降。
2. XOR 单层时指标进入震荡或长期无法归零。
3. XOR 两层网络时损失下降并最终分类成功。

## 5. 交互设计

### 5.1 线性可分场景

功能要求：

1. 可以切换或重置一组固定的线性可分样本。
2. 支持单步训练，观察每次更新后分界线如何移动。
3. 支持自动训练动画，并可暂停。
4. 实时显示当前错误样本和参数变化。

设计原则：

1. 使用固定随机种子或固定初始参数，保证每次演示结果稳定。
2. 动画要足够平滑，但不能牺牲数值真实度。

### 5.2 XOR 单层感知机场景

功能要求：

1. 使用真实 XOR 数据集进行单层感知机训练。
2. 支持单步与自动训练。
3. 明确显示一条线始终无法同时正确分开四个点。
4. 指标面板显示训练无法稳定收敛到正确分类的事实。

讲解重点：

1. 问题不是“还没训练够”，而是“模型只能学一条线”。
2. XOR 需要非线性组合，超出单层线性分类器的表达能力。

### 5.3 XOR 两层网络场景

功能要求：

1. 使用一个小型两层网络对 XOR 数据进行真实训练。
2. 支持单步、自动训练、暂停、重置。
3. 展示隐藏层中间边界与最终分类区域。
4. 训练结束后能稳定给出正确分类结果。

讲解重点：

1. 两层网络不是“画一条更复杂的线”，而是先构造中间特征，再组合成最终分类结果。
2. 通过隐藏层，模型具备了处理非线性可分问题的能力。

## 6. 技术实现设计

### 6.1 接入方式

不新增路由依赖，避免扩大改动范围。采用同一个前端入口，根据 URL 查询参数决定渲染主页面还是独立 Demo 页。

示例方案：

1. 默认渲染当前主页面。
2. 当查询参数满足 `?demo=perceptron` 时，渲染独立感知机 Demo 页。
3. 主页面按钮通过 `window.open` 打开带参数的新窗口。

### 6.2 组件拆分

建议新增并拆分为以下组件：

1. `PerceptronDemoPage`：演示页壳组件，负责布局、场景切换和全局状态协调。
2. `PerceptronPlaneCanvas`：绘制二维坐标、样本点、决策边界、分类区域。
3. `PerceptronControlPanel`：训练控制与参数调节。
4. `PerceptronMetricsPanel`：指标、参数和讲解信息。

如有必要，可继续拆出训练趋势组件，用于底部小图表显示。

### 6.3 绘制方式

优先使用 SVG 绘制坐标系和边界，原因如下：

1. 适合做教学型标注和高亮。
2. 便于响应式布局。
3. 不需要引入额外图形库，便于控制包体和代码复杂度。

### 6.4 状态与训练逻辑

训练逻辑保持真实，不做纯伪动画。

1. 线性可分部分使用经典感知机更新规则。
2. XOR 单层场景同样使用真实单层感知机训练逻辑，并展示其失败过程。
3. XOR 两层场景使用轻量级两层网络进行真实训练，规模控制在演示所需的最小范围内，保证性能与稳定性。

需要维护的核心状态包括：

1. 当前场景。
2. 样本集。
3. 参数矩阵或权重、偏置。
4. 当前 epoch。
5. 自动训练开关。
6. 学习率、训练速度。
7. 训练历史数据。

### 6.5 性能与稳定性

本次改造需要兼顾性能，重点策略如下：

1. 样本数量保持在教学演示所需的最小集合，不引入无意义的大量点。
2. 自动训练时按节流步长推进，避免频繁重绘。
3. 固定初始化策略，避免每次打开演示效果波动过大。
4. 两层网络结构保持轻量，防止浏览器端训练造成卡顿。

## 7. 文案与视觉方向

视觉延续现有深色科技风，但独立 Demo 页会更偏投屏演示风格：

1. 标题更大。
2. 画布更居中。
3. 说明文字更短、更聚焦结论。
4. 页面避免滚动，优先使用完整视窗布局。

主页面文案策略：

1. 不堆公式。
2. 用“学一条线”和“XOR 超出单层能力”建立直觉。
3. 用按钮引导到完整演示。

独立页文案策略：

1. 每个场景只保留一句最关键的讲解。
2. 参数与结果分区展示，避免相互干扰。
3. 强调“几何解释”和“训练过程”并行出现。

## 8. 异常与降级处理

1. 如果新窗口被浏览器拦截，需要在主页面入口附近给出可理解的提示。
2. 如果自动训练中断，仍可使用单步训练完成演示。
3. 如果某个场景未完成训练，界面仍应保持可重置与可继续。

## 9. 验收标准

本次改造完成后，需要满足以下结果：

1. 主页面中的感知机区域相比当前版本更直观地体现平面分类概念。
2. 用户点击按钮后，能在新窗口中打开独立感知机 Demo 页。
3. 独立 Demo 页可以稳定切换三个场景：
   - 线性可分训练成功
   - XOR 单层感知机失败
   - XOR 两层网络成功
4. 训练过程、分界线变化和指标变化能被观众清楚观察。
5. 改造不破坏现有页面其它区域的结构与格式。

## 10. 实施边界

本轮不做以下内容：

1. 不引入新的路由库。
2. 不扩展到更多模型讲解场景。
3. 不为当前需求额外编写大规模单元测试。
4. 不修改与感知机区域无关的章节排版。

## 11. 自检结论

已完成以下自检：

1. 目标与范围一致，没有与用户已确认方案冲突。
2. 主页面与独立 Demo 页分工明确，没有职责混乱。
3. XOR 单层失败和两层成功都被明确纳入实现范围。
4. 技术路径限定为低侵入接入，符合当前项目结构与本次改造边界。
