# 反向传播 Demo 优化设计

## 1. 背景与目标

当前项目中的反向传播 Demo 以“信号流动动画”为主，更适合做抽象概念说明，但不足以把“误差如何回传”和“参数如何更新”讲透。此次改造目标与感知机 Demo 的优化经验保持一致，采用“主页面轻讲解 + 独立反向传播教学页”的分层方案。

本轮改造需要达成两个目标：

1. 主页面中的“单层到多层，再到反向传播”区域保持简洁，只负责建立直觉和提供入口。
2. 独立教学页完整展示反向传播过程，既能看见链式法则如何回传误差，也能看见参数如何依据梯度更新。

## 2. 用户已确认范围

已确认需求如下：

1. 采用“主页面轻讲解 + 独立反向传播教学页”方案。
2. 独立页需要同时讲解两类核心过程：
   - 链式法则下误差如何逐层回传。
   - 参数如何根据梯度进行更新。
3. 独立页需要支持两种网络规模：
   - 极简网络。
   - 稍大网络。
   - 两者可切换。
4. 独立页需要支持两种训练驱动方式：
   - 单样本逐步演示。
   - 按 epoch 演示。
   - 两者可切换。

## 3. 推荐方案

### 3.1 主页面

主页面中的反向传播区域只做轻量入口改造，不改变其它章节格式。

改造后的内容包含：

1. 一张简洁网络示意图，说明“前向得到预测与损失，反向将误差传回，最后更新参数”。
2. 简短文案，明确区分“误差回传”和“参数更新”。
3. 一个“打开完整 Demo”按钮，使用新窗口进入独立教学页。

### 3.2 独立教学页

独立页作为完整教学空间，承载反向传播过程的细化展示。页面必须同时服务两个视角：

1. 结构和流向视角：看误差信号如何沿网络反向传播。
2. 数值和更新视角：看 loss、梯度、权重和偏置如何变化。

## 4. 页面结构设计

### 4.1 顶部控制栏

顶部控制栏用于统一控制教学流程，至少包含：

1. 网络规模切换：极简网络 / 稍大网络。
2. 训练模式切换：单样本模式 / 按轮模式。
3. 播放控制：单步、自动播放、暂停、重置。
4. 可调参数：学习率、播放速度。

### 4.2 中央主画布

主画布负责展示网络结构与传播路径，重点包括：

1. 节点、边、层级结构。
2. 前向传播时的激活值、权重与预测值。
3. 损失产生位置与输出误差。
4. 反向传播时误差路径高亮。
5. 每条边或每个节点的梯度信息。
6. 参数更新前后的变化高亮。

### 4.3 右侧讲解与指标区

右侧区域用于配合主画布进行文字和数值讲解，包含：

1. 当前步骤说明。
2. 当前样本或当前 epoch 信息。
3. 当前预测值、目标值、loss。
4. 当前梯度、权重、偏置变化。
5. 对链式法则或当前更新的简短教学提示。

### 4.4 底部过程区

底部过程区根据模式切换不同内容：

1. 单样本模式下展示步骤时间线：
   - 前向计算
   - 计算损失
   - 输出层梯度
   - 隐藏层梯度
   - 参数更新
2. 按轮模式下展示训练趋势：
   - loss 曲线
   - 准确率变化
   - 梯度大小变化

## 5. 交互模式设计

### 5.1 单样本模式

单样本模式用于讲透一次完整 backprop 过程。每次点击单步，推进一个明确阶段，而不是直接跳完整轮训练。

典型步骤为：

1. 前向传播计算隐藏层与输出层结果。
2. 基于预测值与目标值计算损失。
3. 计算输出层误差与梯度。
4. 将误差信号传回隐藏层。
5. 根据梯度更新参数。

此模式的核心目标是帮助观众理解：

1. 误差不是凭空出现，而是从损失出发。
2. 每一层的梯度来自后续误差与本层局部导数的组合。
3. 参数更新是反向传播的结果，而不是独立步骤。

### 5.2 按轮模式

按轮模式用于观察整体训练趋势，而不是逐行讲步骤。

此模式重点展示：

1. 每个 epoch 后的整体 loss 变化。
2. 网络预测逐渐逼近目标的过程。
3. 梯度和参数更新累积后的训练效果。

此模式的核心目标是帮助观众理解：

1. 反向传播不是一帧动画，而是支撑训练收敛的机制。
2. loss 的下降和预测的改善来自多次梯度更新积累。

### 5.3 网络规模切换

网络规模支持两档：

1. 极简网络：优先用于讲清每个节点、每条边和每个梯度。
2. 稍大网络：用于展示结构稍复杂时，误差如何在多条路径上传播。

切换网络规模时，应同步重置当前训练状态，避免混淆。

## 6. 技术实现设计

### 6.1 接入方式

沿用当前已验证的独立 Demo 接入方式，不新增路由库。

通过查询参数区分渲染内容，例如：

1. 默认渲染当前主页面。
2. 当查询参数为 `?demo=backprop` 时，渲染独立反向传播教学页。
3. 主页面按钮通过 `window.open` 打开新窗口。

### 6.2 组件拆分

建议拆分为以下组件：

1. `BackpropDemoPage`：独立页壳组件，负责布局、模式切换和整体状态协调。
2. `BackpropNetworkCanvas`：负责绘制网络结构、激活值、误差路径、梯度和参数变化。
3. `BackpropControlPanel`：负责网络规模切换、训练模式切换、播放控制和参数调节。
4. `BackpropMetricsPanel`：负责当前样本、loss、梯度、参数更新和教学说明。
5. `BackpropTimelinePanel`：负责单样本步骤时间线或按轮趋势展示。

### 6.3 状态设计

核心状态至少包括：

1. 当前网络规模。
2. 当前训练模式。
3. 当前步骤阶段。
4. 当前样本索引或当前 epoch。
5. 当前权重、偏置、激活值。
6. 当前 loss、梯度与更新结果。
7. 自动播放状态与播放速度。
8. 当前教学日志或步骤记录。

### 6.4 训练与演示逻辑

训练逻辑需要兼顾教学可控性与过程真实感：

1. 单样本模式应按步骤拆解一次完整 backprop。
2. 按轮模式应按完整训练轮推进，并记录整体趋势。
3. 网络规模控制在教学所需的最小范围内，避免过度复杂。
4. 使用固定初始化策略，保证演示稳定。

### 6.5 可视化实现

优先使用 SVG 绘制网络结构和路径，原因如下：

1. 便于按节点和边进行精细高亮。
2. 适合显示权重、梯度、方向箭头和步骤标注。
3. 不需要引入额外图表库，控制改动和包体。

## 7. 主页面改造设计

主页面中的“单层到多层，再到反向传播”区域改造重点如下：

1. 不再只用三张说明卡表达。
2. 新增一张简洁网络图，说明前向与反向传播的关系。
3. 用一小段文案解释：
   - 前向负责算预测和损失。
   - 反向负责把损失对参数的影响逐层拆开。
4. 增加“打开完整 Demo”按钮，引导进入独立教学页。

## 8. 视觉方向

视觉沿用当前深色科技风，但强调“教学可读性”：

1. 当前路径高亮要明显。
2. 前向与反向的颜色区分要清楚。
3. 数值、步骤和结构不能相互抢信息。
4. 主页面保持简洁，独立页承担完整讲解。

## 9. 实现边界

本轮不做以下内容：

1. 不将其扩展为通用神经网络训练平台。
2. 不支持任意自定义层数和任意网络结构输入。
3. 不引入复杂优化器、正则化、批标准化等额外主题。
4. 不修改与反向传播区域无关的其它章节排版。

## 10. 验收标准

改造完成后，需满足以下结果：

1. 主页面中的反向传播区域更适合讲解，并具备清晰入口。
2. 点击按钮能在新窗口打开独立反向传播教学页。
3. 独立页支持网络规模切换。
4. 独立页支持单样本模式与按轮模式切换。
5. 单样本模式可清楚展示一次完整的前向、损失、反向与更新流程。
6. 按轮模式可展示整体 loss 和训练趋势。
7. 现有其它 Demo、测试和构建流程不受破坏。

## 11. 自检结论

已完成以下自检：

1. 设计与用户确认的“主页面轻讲解 + 独立教学页”方案一致。
2. 链式法则过程和参数更新过程均已纳入设计范围。
3. 网络规模切换与训练驱动方式切换均有明确落点。
4. 接入方式沿用当前项目已验证模式，改动边界清晰。
